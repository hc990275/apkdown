TVBox APK 自动化更新脚本说明
本脚本专为 Android 环境（推荐在 MT 管理器拓展包下运行）设计，旨在实现多套 TVBox 影视应用 APK 的版本自动化检测与下载更新。以下是脚本的功能特性、操作指南及维护结构。
✨ 核心功能亮点
 * 脚本自更新机制：自动与 hc990275/apkdown 仓库比对，若有新版本脚本，将自动下载替换并提示用户重新运行，确保您使用的永远是最新版本。
 * 多版本同步监控：
   * OK 版：支持手机/电视/Pro 版本。
   * 蜜蜂版：支持手机/电视版本。
   * 通过拉取各渠道的 JSON/版本号文件，精准判断版本变更。
 * 全系 APK 批量下载：内置详细的 APK 直链映射表，涵盖 32/64 位、海信专版、安卓 4.0 旧版及 Pro 变体等，确保版本更新时，所有对应包都能一次性批量下载。
 * 环境与权限校验：启动时会检查 MT 管理器拓展包环境，并提示用户确保对下载目录拥有读写权限。
 * 可视化结果报告：执行完毕后，将以随机彩色输出的形式清晰展示更新摘要和下载结果，并附带公众号提示信息。
🛠️ 脚本执行流程
 * 启动检查：显示当前脚本版本，并进行 MT 管理器拓展包环境检测。
 * 自我更新：调用 GitHub API 检查并下载最新版脚本。
 * 版本比对：遍历配置中的所有版本信息 URL，下载临时文件，并与本地缓存的旧版本进行比对。
 * 执行下载：若检测到任一渠道版本号有变更，立即更新本地缓存文件，并依据预设的 APK_PATHS 规则批量下载对应的 APK 文件。
 * 完成报告：终端输出版本汇总报告、公众号信息，标记脚本执行结束。
📂 关键目录结构
| 目录/文件 | 描述 |
|---|---|
| 下载目录 | /storage/emulated/0/0网站/下载专用/影视安装包更新 |
| 版本文件夹 | ${下载目录}/版本文件夹 (用于缓存各渠道的 JSON/版本号文件，供比对使用) |
| APK 文件 | 成功下载的 APK 将以“名称.apk”的形式直接保存在 下载目录 |
💡 使用建议与维护
运行环境要求
 * 网络条件：需确保设备可以正常访问 GitHub Raw 文件 和 GitHub API。脚本内部使用 wget 命令进行下载。
 * 运行方式：强烈建议在 MT 管理器拓展包 环境下执行，以获得最佳兼容性及权限保障。
 * 权限设置：必须确保脚本对 下载目录 及 版本文件夹 拥有完全的读写权限。
异常处理
 * 若在下载某个 JSON 文件或 APK 时失败，终端会输出对应的错误提示。用户只需重复运行脚本即可重试未完成或失败的下载任务。
